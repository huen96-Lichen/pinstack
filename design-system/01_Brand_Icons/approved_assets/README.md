# PinStack Approved Icon Assets

Frozen in `v2.4.9`.

Current confirmed grouping:

- `app-icon`
  - Source: `pinstack-app-icon-master.jpg`
  - PNG: `pinstack-app-icon-master.png`
  - Role: brand recognition

- `menubar-icon`
  - Source: `pinstack-menubar-icon.svg`
  - PNG sizes: `png/menubar-icon/`
  - Role: extreme small-size recognition

- `floating-button`
  - Source: `pinstack-floating-button-icon.svg`
  - PNG sizes: `png/floating-button/`
  - Role: desktop entry experience

Decision notes:

- `app-icon` keeps the approved pin + stacked cards master image.
- `menubar-icon` follows the current floating launcher language, but stays monochrome and compact.
- `floating-button` keeps the current in-product launcher style and is not forced into an app-icon mini version.

Release mapping:

- `app-icon` -> `assets/icons/app`
- `menubar-icon` -> `assets/icons/tray`
- `floating-button` -> `assets/icons/floating-button`
