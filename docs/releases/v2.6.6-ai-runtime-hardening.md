# PinStack v2.6.6 AI Runtime Hardening（AI 体验优先 + 高质量优先）

- 发布时间：2026-04-18
- 版本目标：把 AI Hub、独立聊天窗口、主进程执行层收拢到统一可执行/可诊断/可回退的执行体系。

## 1. 本版重点

- 云端执行从占位切换为真实 OpenAI 路径，默认走高质量策略。
- API Key 从设置明文迁移到 macOS Keychain，设置文件仅保留配置态。
- 新增统一路由治理：任务类型 -> 策略 -> provider/model -> timeout/retry -> output target。
- 错误语义标准化并可恢复：`AUTH_INVALID`、`RATE_LIMIT`、`TIMEOUT`、`MODEL_UNAVAILABLE`、`NETWORK_UNREACHABLE`、`RUNTIME_NOT_READY`。
- 可观测性收口：路由选择、任务开始/成功/失败、回退触发、时延桶都可追踪。

## 2. 主要变更

### 2.1 执行核心

- `AiHubService` 增加真实云端调用能力，支持请求 ID、错误码、时延回填。
- AI 聊天与健康检查/测试统一返回增强字段（provider/requestId/errorCode/latencyMs）。
- 云端失败时按设置触发本地回退，不再静默成功。

### 2.2 安全与配置

- 主进程新增 `secretStore`，通过 macOS `security` CLI 读写 Keychain。
- 启动迁移与设置更新都走密钥拦截：旧明文自动迁入 Keychain 并清理配置。
- `SettingsService.persist` 强制清洗 `aiHub.cloudApiKey`，杜绝回写明文。

### 2.3 路由统一

- 新增 `taskRouter` 作为编排任务路由真相源，统一维护：
  - `taskRequiresRecord`
  - `outputTarget`
  - `timeoutMs`
  - `retryLimit`
  - `strategy`
- `ai.orchestrate.run` 接入 `taskRouter`，避免分支硬编码分叉。

### 2.4 诊断与 IPC

- IPC 扩展（兼容）：
  - `ai.config.migrateSecrets`
  - `ai.diagnostics.snapshot`
  - `ai.health`（兼容别名，保持 `ai.healthCheck` 可用）
- 设置页专家模式新增诊断快照展示与密钥迁移/刷新按钮。

## 3. 回归结果

- `npm run typecheck`：通过
- `npm run test`：通过（154/154）
- `npm run dev`：可启动，主进程日志可见 7 项快捷键注册成功与 AI 运行日志。

## 4. 风险与后续

- 云端链路的真实鉴权/限流/网络异常，仍建议在目标环境做一次人工回归。
- Anthropic 多 provider 路由暂未纳入本版，后续版本增量实现。
