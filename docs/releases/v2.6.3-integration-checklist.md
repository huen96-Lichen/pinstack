## 0. 前置声明（必读）

- [ ] 当前 **Cloud Provider 仍为 Mock 占位**，不是正式云端能力（不得按生产云端能力验收）。
- [ ] AI 能力是增强层，不是主流程硬依赖：关闭 AI 后主流程（捕获/收藏/管理）仍可用。
- [ ] 本次验收目标为功能可见性、可配置性、联调可用性，不等同于最终云端生产能力验收。

---

## 1. AI 入口与可见性

### 1.1 控制面板一级 AI 入口
- [ ] 控制面板可直接看到 AI 一级入口（AI 助手/AI 中枢）。
- [ ] 点击 AI 一级入口可进入 AI 使用视图（不是只能跳设置页）。
- [ ] AI 入口文案与状态可读（可看出是否启用、当前 provider/model）。

### 1.2 右上角 AI 入口
- [ ] 右上角 AI 图标可稳定显示（按显示策略）。
- [ ] 点击右上角 AI 图标可打开轻量对话面板。
- [ ] 对话面板中可看到当前执行 provider/model。

### 1.3 AI 入口显示策略
- [ ] `always`：无论是否启用 AI，入口都显示。
- [ ] `enabled_only`：仅启用 AI 时显示入口。
- [ ] `hidden`：入口隐藏，且导航不会误落到 AI 视图。

---

## 2. AI-first 搜索

- [ ] 设置中可看到 AI-first 搜索开关。
- [ ] 开启 AI-first 后，搜索会触发 AI 意图解析（source/type/tags 有辅助变化）。
- [ ] 关闭 AI-first 后，搜索行为回到原始搜索逻辑。
- [ ] AI 总开关关闭时，AI-first 不应影响普通搜索可用性。

---

## 3. 收藏页 AI 整理素材库

- [ ] 收藏页存在“AI 整理素材库”入口。
- [ ] 可执行“预览 -> 应用”流程（不应无预览直接大规模改写）。
- [ ] v1 动作可验证：统一命名、一级分类、主关键词、可选摘要。
- [ ] `suggestionOnly` 开启时仅建议不应用。
- [ ] 应用后用户手改标题不被覆盖（标题保护生效）。
- [ ] 摘要为旁路增强，不阻塞主流程。
- [ ] dedupeSuggestion 仍为建议模式，不自动删除/自动改主记录。

---

## 4. AI / 飞书设置分离

- [ ] 设置中 AI 与飞书是独立承载区（非混杂在同一块）。
- [ ] AI 区可配置：总开关、provider、model、入口策略、AI-first、persona 槽位。
- [ ] 飞书区可独立查看连接状态并手动检查。
- [ ] 调整 AI 设置不应破坏飞书配置；调整飞书不应改写 AI 配置。

---

## 5. Provider / Model 切换一致性

- [ ] 可在设置中切换默认 provider（local / cloud）。
- [ ] provider 切换后，UI 显示与实际执行源一致（配置态与实际态一致）。
- [ ] model 选择来自受控 registry，不允许自由输入任意模型名直接启用。
- [ ] local provider 不可达/模型缺失时，状态与 fallback 原因可见。
- [ ] cloud provider 当前明确显示 mock 占位，不可误解为正式云端。

---

## 6. Persona Markdown 注入（3 槽位）

- [ ] 设置中可看到最多 3 条内置 markdown persona/规则槽位。
- [ ] 每个槽位支持启用/停用。
- [ ] UI 可看到当前实际启用的规则槽位。
- [ ] 发送 AI 对话/执行任务时能观察到规则注入效果（风格或约束变化）。
- [ ] 未启用的槽位不应参与注入。

## 7. DMG 安装与启动验证（本机）

- [ ] 产物存在：`PinStack-2.6.3-arm64.dmg`
- [ ] DMG 可挂载并安装到 Applications。
- [ ] App 可启动，版本显示为 `2.6.3`。
- [ ] 首次打开后可看到 AI 入口（按默认策略与启用状态）。
- [ ] 设置页可打开，AI 与飞书分离区可见。

## 8. 另一台电脑冷启动实验（重点）

> 目标：验证“换机首次安装 + 无开发环境 + 冷启动”可联调可理解。

### 8.1 安装与启动
- [ ] 在另一台 Mac 上从 DMG 安装并首次启动成功。
- [ ] 无需开发命令即可进入主界面（非开发依赖启动）。
- [ ] 版本显示正确：`2.6.3`。

### 8.2 首次可见性
- [ ] 新机器首次打开即可感知 AI 能力入口（符合显示策略）。
- [ ] 右上角 AI 对话入口可打开，不崩溃。
- [ ] 设置中 AI 与飞书分离可见，文案可理解。

### 8.3 冷启动稳定性
- [ ] 首次打开 AI 对话面板可正常发送与回显。
- [ ] 关闭 App 后再次冷启动，AI 对话历史可回读（单会话持久）。
- [ ] AI-first 搜索开关状态在重启后保持一致。

### 8.4 环境差异验证
- [ ] 若本地模型未安装/不可达，系统给出清晰状态与降级提示。
- [ ] 即便 AI 不可用，主流程（收藏/管理/捕获）仍可正常使用。
- [ ] Cloud 入口仍明确为 mock 占位，不误导为真实云端联通。
