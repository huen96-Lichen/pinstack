# PinStack v2.5.4 修复说明

## 修复范围
- 截图实际可用但首页仍持续提示“需要权限”
- 打包版启动后偶发空白白屏

## 根因
### 1. 权限阻塞误判
- `v2.5.3` 已经把截图可用性改成以真实 `screencapture` 链路为准。
- 但 UI 侧仍然把 `needsAttention` 当成首页阻塞条件。
- 结果是：即使截图已经可用，只要某个权限项仍处于“建议刷新/建议重启”状态，首页仍会继续显示“需要权限”。

### 2. 打包启动白屏
- renderer 顶层仍使用 `lazy + Suspense`。
- fallback 是一个透明空白壳。
- 在打包环境中一旦顶层 chunk 加载延迟或异常，用户会看到长时间停留的空白白屏。

## 修复内容
### 权限链路
- 新增 `blocking` 字段，用于区分：
  - 会真正阻塞主流程的问题
  - 仅用于诊断的提示项
- 新增 `hasBlockingIssues`
- 首页权限提示和顶部状态芯片现在只看 `hasBlockingIssues`
- 当截图链路已可用时，屏幕录制权限直接视为可用，不再继续触发阻塞提示

### 启动链路
- 移除 renderer 顶层 `lazy + Suspense`
- `Dashboard / PinView / CaptureOverlay / CaptureHub / CaptureLauncher` 改为直接导入
- 避免打包启动时空白 fallback 被长时间停留
