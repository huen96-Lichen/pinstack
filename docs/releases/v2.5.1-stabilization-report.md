# PinStack v2.5.1 修复说明（权限链路稳定化 + 系统集成修复）

发布日期：2026-04-04  
基线版本：v2.5.0  
目标版本：v2.5.1

## 1. 本次修了哪些问题

### Bug 1：系统里已开启权限，但 PinStack 仍显示未授权
- 将权限状态拆成独立项，不再用单一 `hasPermission=false` 覆盖多个问题。
- 将权限提示统一为结构化快照：每项包含状态、最近检查时间、说明、建议动作、是否可刷新、是否可打开系统设置。
- 将权限刷新接入真实刷新链路，而不是只重绘 UI。
- 在 UI 中加入实例不一致提示与权限诊断信息，减少“明明开了权限却还显示未授权”的误导。

### Bug 2：菜单栏图标显示为白色方块
- 修复 tray icon 资源查找与加载逻辑。
- 优先加载单色 template PNG 与 `@2x`，并在 macOS 上强制标记为 template image。
- 当资源异常时输出清晰日志，并降级到备用图标，不再静默出现白块。

### Bug 3：Dock / Finder / Cmd+Tab 没有显示正式 App Icon
- 将正式打包图标链路统一到 `assets/icons/app/pinstack-app-icon.icns`。
- 补齐运行时 Dock 图标应用逻辑与诊断日志。
- 保持 Dock / Finder / Cmd+Tab 使用同一套彩色 App Icon 资源，与 tray template 图标分离。

### Bug 4：`Command + Shift + V` 弹出历史遗留小窗口
- 正式下线该历史遗留快捷键入口。
- 删除旧搜索小窗主链路，不再保留隐藏式残余逻辑。

## 2. 根因分析

### Bug 1 根因
- 权限模型过于扁平，屏幕录制、辅助功能、自动化依赖被混成笼统“未授权”。
- 授权后的刷新时机不统一，返回应用后仍可能停留在旧状态。
- 缺少当前运行实例的路径、bundleId、dev/prod 诊断信息，难以判断是不是“授权的是 A，运行的是 B”。

### Bug 2 根因
- tray icon 资源链路不够稳，资源缺失或加载失败时没有按 macOS template 图标规则兜底。

### Bug 3 根因
- 正式 App Icon 的 `.icns` 链路没有完全固定到打包配置与运行时诊断路径。
- Dock / Finder / Cmd+Tab 使用的是系统级图标链路，不能依赖前端普通 PNG 或局部窗口图标。

### Bug 4 根因
- 历史版本遗留的 Quick/Search 小窗入口没有彻底退役，快捷键与窗口接线仍残留在代码里。

## 4. 验证结果

### Bug 1：权限链路
**如何验证**
- 启动应用，查看权限状态卡、顶部状态入口、设置页诊断区是否显示独立权限项。
- 打开系统设置修改屏幕录制或辅助功能权限。
- 返回 PinStack 后触发自动刷新，或点击“刷新状态”。
- 查看日志中是否输出：应用路径、可执行文件、bundleId、dev/prod、最近检查来源。

**验证结果**
- 独立权限项已分开显示，不再把多类问题混成“未授权”。
- 刷新入口已调用主进程真实检测逻辑，不是 UI 假刷新。
- UI 侧已能显示实例不一致提示与最近检查时间。

**仍需线下确认**
- 真实系统授权后返回应用时的最终肉眼体验，需要在目标 macOS 机器上完成一次人工验收。

### Bug 2：菜单栏图标
**如何验证**
- 启动应用，观察顶部菜单栏图标。
- 检查控制台 `[tray.icon]` 日志，确认命中 template PNG 或 fallback。
- 在深色/浅色菜单栏下观察是否仍出现白块。

**验证结果**
- tray icon 资源链路、template 标记、fallback 与日志已补齐。
- 白块问题已从“静默失败”变为“可定位、可兜底”的明确链路。

**仍需线下确认**
- 深浅模式与 Retina 屏下的最终视觉效果，需要在真实菜单栏环境中人工复验。

### Bug 3：Dock / Finder / Cmd+Tab 图标
**如何验证**
- 检查 `package.json` 打包配置是否指向 `assets/icons/app/pinstack-app-icon.icns`。
- 运行打包后，检查 `.app` / DMG 产物。
- 在 Dock、Finder、Cmd+Tab 中查看图标是否统一。

**验证结果**
- 打包配置与运行时 Dock 图标候选路径已统一到正式 `.icns`。
- 日志链路已补齐，可以明确看到 dev/prod 命中的图标路径。

**仍需线下确认**
- Dock / Finder / Cmd+Tab 的最终显示仍需在正式打包产物上做肉眼验收，并排除系统缓存影响。

### Bug 4：历史遗留快捷键窗口
**如何验证**
- 按 `Command + Shift + V`，确认不再弹出旧搜索小窗。
- 搜索代码中是否还存在 `QuickPanel`、旧窗口创建链路和该快捷键注册。

**验证结果**
- 快捷键已退役，旧窗口与渲染页已删除，代码层正式下线。

## 5. 仍未解决的问题
- `v2.5.1` 已完成代码链路修复，但以下仍需在真实目标机做最终人工验收：
  - 系统授权后返回 PinStack 的状态刷新是否符合预期
  - 菜单栏图标在深浅模式与 Retina 屏下的最终视觉表现
  - 正式打包产物在 Dock / Finder / Cmd+Tab 中的图标一致性
- 这些属于最终发布前的系统环境验收，不再是代码结构缺口。
