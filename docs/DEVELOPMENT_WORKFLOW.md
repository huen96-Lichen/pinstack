# PinStack Development Workflow (Codex 协作规范)

## 1. Task Execution Rules

- 一次只执行一个 Step。
- 每完成一个 Step 必须停止并等待确认。
- 未获确认不得进入下一 Step。
- 禁止跨阶段修改与“提前实现后续需求”。

## 2. Modification Constraints

- 未明确要求时，不修改数据结构。
- 未明确要求时，不修改 IPC 协议。
- 不新增依赖。
- 不做大规模重构。
- 优先最小改动实现目标。

## 3. Output Requirements

每次任务完成后必须输出：

1. 修改文件列表。
2. 修改内容说明（按目标对应）。
3. 未修改范围说明（安全边界）。
4. 本机验证步骤（可复现）。

## 4. Testing Workflow

- 必须进行本地运行验证（至少 `typecheck` 与 `build`）。
- 必须执行关键路径手动验证。
- 验证结果需说明“通过/失败 + 失败原因”。
- 若无法验证，必须明确阻塞条件与最小补救方案。

## 5. Change Scope Control

- 禁止“顺手优化”。
- 禁止 UI/逻辑扩散修改。
- 仅在任务范围内改动，超范围项进入下一任务单独执行。
- 发现潜在问题可记录，但不得在当前任务直接扩展实现。
