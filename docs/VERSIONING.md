# PinStack Versioning

## Versioning Strategy

PinStack 采用 `MAJOR.MINOR.PATCH`（Semantic Versioning）：

- `MAJOR`：产品阶段或定位变化，涉及架构主线或核心心智模型升级。
- `MINOR`：在当前主线下新增可见能力，不破坏既有兼容预期。
- `PATCH`：缺陷修复、稳定性优化、体验优化，不引入阶段性能力变化。

## Upgrade Rules

### MAJOR 升级条件

- 产品定位从一个阶段跨入下一个阶段（例如 `2.x -> 3.x`）。
- 核心工作流或数据/交互主范式发生不可忽略的升级。

### MINOR 升级条件

- 在当前阶段引入新功能模块，且对用户可感知。
- 新能力属于“新增”，不是“修补”。

### PATCH 升级条件

- 修复缺陷、性能优化、交互细化、稳定性收敛。
- 不改变产品阶段，不引入新的核心能力面。

## Usage Guidelines

- 先判断“是否跨阶段”，再决定 `MAJOR`。
- 不跨阶段但新增能力，使用 `MINOR`。
- 仅修复与优化，使用 `PATCH`。
- 每次版本变更必须同步更新 `CHANGELOG.md`，并与实际交付一致。
- 发布前需完成最小验证闭环：`typecheck + build + 关键手动路径`。
